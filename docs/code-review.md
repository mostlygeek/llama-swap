# Code Review Guidelines

- Use three severity levels: High, Medium, Low
- Label each issue with a tag like H1, M2, L3
- High: must fix (security, race conditions, critical bugs)
- Medium: recommended improvements (coding style, missing functionality, inconsistencies)
- Low: nice to have changes and nits
- Include a suggestion with each discovered item
- Limit your code review to three items with the highest priority first
- Double check your discovered items and recommended remediations
